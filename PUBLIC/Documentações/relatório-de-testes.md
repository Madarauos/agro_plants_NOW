**Escopo de Testes**

Objetivo:
Definir o que será testado no backend e no frontend para identificar e corrigir bugs antes da entrega da nova versão.

Tipo de lançamento:
Versão de manutenção com correção de bugs e pequenas melhorias no fluxo.

Funcionalidades incluídas no teste:
Front-end:
Landing-page
Sobre nós e Contate-nos
Login e logout de usuários.
Cadastro de novos usuários.
Navegação e exibição de produtos.
Telas de Ajuste.
Lista de Clientes e Vendedores.
Cadastro de Vendas.
Catálogo.
Dashboard.
Relatório de vendas.
Cadastro de Cupons
Responsividade em dispositivos móveis e desktops.
Cards.

Pop-up:
Cadastrar vendedor, vendas, produtos, serviços e clientes.
Remover e Adicionar Cupons.
Remover vendedor, produtos, serviços e clientes.
Cadastrar comissão.
Criar senha.
Recuperar email e senha.
Alterar senha.
Perfil adm.
Link do whatsapp.

Backend:

API de autenticação (login/cadastro).
API de listagem e busca de produtos.
API de Cadastros Gerais.
API comissão.
Integração com banco de dados.

Áreas de risco:
Falha no login, impedindo acesso de usuários.
Botões não funcionais.
Links errados.
Mal integração no banco de dados.
Responsividade.

Histórico de regressões:
Padronização.
Menu com bugs.
botões não funcionais.
links errados.


Restrições e dependências:
Dependência do ambiente de testes estar estável.

Critérios para considerar “concluído”:
Todos os casos de teste executados e documentados.
Nenhum bug crítico pendente.
Bugs médios e baixos registrados no sistema de issues.
